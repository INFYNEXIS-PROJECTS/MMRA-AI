Thank you for your interest in contributing to MMRA-AI
MMRA-AI is an AI-focused collaborative project under the INFYNEXIS-PROJECTS organization. We welcome developers, designers, researchers, technical writers, and contributors of all experience levels.
Our goal is to build scalable, maintainable, and innovative AI-powered systems through collaboration and clean engineering practices.

# GETTING STARTED

## 1. Fork the Repository
Click the Fork button on the top-right corner of this repository.

## 2. Clone Your Fork
git clone https://github.com/YOUR_USERNAME/MMRA-AI.git

## 3. Navigate Into the Project
cd MMRA-AI

## 4. Install Dependencies
npm install

## 5. Start the Development Server
npm run dev

# DEVELOPMENT SERVER
Before contributing, ensure you have:
- All requirements installed (check requirements.txt for list)
- Git installed
- A modern code editor such as VS Code
- Basic understanding of python and streamlit

# BRANCHING STRATEGY
Never commit directly to the `main` branch.
Create a separate branch for every feature, fix, or update.

## Branch Naming Convention
text
feature/feature-name
fix/bug-name
docs/documentation-update
refactor/module-name

### Examples
text
feature/add-auth-system
fix/sidebar-overflow
docs/update-readme
refactor/dashboard-components

# CONTRIBUTION WORKFLOW
## Standard Workflow
1. Fork the repository
2. Clone your fork
3. Create a new branch
4. Make your changes
5. Test your implementation
6. Commit your changes
7. Push to your fork
8. Open a Pull Request

# CODING STANDARDS
## General Rules
- Write clean and readable code
- Keep components modular
- Avoid unnecessary complexity
- Use meaningful variable and function names
- Follow the existing folder structure
- Remove unused imports and console logs before committing

# COMMIT MESSAGE CONVENTION
Please follow conventional commit formatting.

## Format
text
type: short description

## Examples
text
feat: add authentication flow

fix: resolve navbar responsiveness issue

docs: update contributing guide

refactor: optimize dashboard rendering

# PULL REQUEST GUIDELINES
Before opening a Pull Request:
- Ensure the project builds successfully
- Test your changes properly
- Resolve merge conflicts
- Update documentation if necessary
- Keep pull requests focused and concise

# REPORTING BUGS

When reporting bugs, please include:
- Clear bug description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable
- Browser/device information

# SUGGESTING FEATURES
Feature suggestions are welcome.
Please provide:
- The problem being solved
- Why the feature is useful
- Possible implementation ideas
- Relevant screenshots/mockups if applicable

# SECURITY
Please do NOT publicly disclose security vulnerabilities.
If you discover a security issue, contact the maintainers privately.

# COMMUNITY GUIDELINES
We are committed to maintaining a respectful, inclusive, and collaborative environment.

Please:
- Be respectful
- Give constructive feedback
- Help other contributors
- Communicate professionally

Toxic behavior, harassment, or discrimination will not be tolerated.

# Questions?
If you need help or clarification:
- Open an issue
- Start a discussion
- Contact the maintainers
